Personal Review of the Webiste


2.1 Review the ongoing use of IT tools and change approach as needed – in making your own website, are you happy with the tools you used? E.g. Visual Studio Code, Codepen, Trello

So far the tools used in this project are:
-Visual Studio Code,
-Trello,
-Slack,
-Zoom,
-Notepad,
-google chrome search engine,
-Duckduckgo search engine,

and by the end GITHUB will be used to upload this project to the internet for review.


Visual Studio Code has been very useful for writing out the code and quickly scanning for
functional mistakes. The primary drawback so far has been that it does not possess a spell
check or correction feature, so with a society that has become intimately used to
having their spelling checked for them it can be difficult to effectively review ones
own work effectively with a skill that may not have been used for many years.
The Notepad software was trialed for the purpose of reliable spelling but it
proved equally useless in this field.
MS Office-word was considered but was not within the budget for
this project to purchase.

Trello was not initially considered useful for this individual project but
but was utilised as it was part of the marking criteria.
Though suprisingly it did serve the purpose in aid of chunking the project into more
manageable parts via it's checklist functtion.

Slack and Zoom have been increadibly useful for communicating between
members of the August 17th 2020 class of Code Nation and communicating with the
illustrious "Arch-tutor Jay : Grand Magnus of the The Code, esteemed capain of
the Code Nation special forces and tamer of the dreaded Python!" praise be his name.

The Google Chrome and DuckDuckGo search engines were used throughout the
testing and trouble shooting proccess during the projects developement with further plans
being made to also test the projects functionality on "mozilla firefox" and
soemthing called a "Yahoo".



2.2 Describe whether the IT tools selected were appropriate E.g. Visual Studio Code, Codepen, Trello

Visual Studios Code seems appropriate for the given task baring the lack
of a spellcheck, though even if there was a tool with such a feature it would not
make it more appropriate to use as the most critically strained resource for this
project was the time constraint.

The same logic can be allplied to the other it tools used.
Due to the inexperience of the developer of this proejct the tools used were the ones
that were made familliar during the compact 3 week course.
To sacrifice time to both; research for a more appropriate tool and learn how to use it,
would have been irresponsible considering both the time constraints and acceptable
degree of function of the currently familiarised IT tools.



2.3 Assess the strengths and weaknesses of your final project – are you happy with your website? Why? Why not? You should review it on the website itself in the interests of openness. What would you change?

The Strengths; of the final product is it's simplicity, the majority of
the alterations made to the style have been for readability and ease of use.

The weakneses; of the final product could also be it's simplicity though...
As standards have changed over the years many if not most browsers of the internet
have come to expect pretty visuals and may often forgo the use of websites without them.
if further proof of this statement is required simply imagine how often the average user
of the internet uses google scholar to learn something compared to the regular google
browser and you'll likely find yourself agreeing.

Am I happy with the website?
No, i'm rarely happy with my work,
especially when prolonged fatigue and personal life have intruded on the quality
of the final product, but it is sadly the current best outcome possible from 5 hours
of focused work at my current ability. Uncertain where but it is believed a couple
of bugs may have entered the code at some point resulting in some lines of code
not working as intended, one fault required a manuel re-writing to remove in
Visual Studios Code but if there is another then it currently lays undetected
causing a significant adaptaion of the intended style of the website.

What would i change?
If more distraction free time was possible to dedicate to this project then the
original plan of creating a parody like old-fashioned Newspaper style website would
have been persued utilising the new and exciting flex-boxes within CSS to re-create
the old tabloid style layout. Some of the original parody text was left within the
website to meet the requirement of adding some personal personality to the website.
